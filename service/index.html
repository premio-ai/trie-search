Introduction

Last updated: March 14, 2026 · Edit this page

Nexus UI is a comprehensive, accessible component library for building modern web applications. It provides a robust set of pre-built components, a flexible theming system, and seamless TypeScript integration out of the box.

💡
If you're new to Nexus UI, we recommend starting with the Quick Start guide to get up and running in under 5 minutes. For migration from v3, see the migration guide.
🎨

Themeable

Fully customizable design tokens with CSS variables. Light and dark mode built-in. Create unlimited custom themes.

♿

Accessible

WAI-ARIA compliant. Full keyboard navigation. Screen reader tested. WCAG 2.1 AA conformance across all components.

⚡

Performant

Tree-shakeable. Zero runtime CSS-in-JS. Lazy-loaded components. Average bundle impact under 8KB gzipped per component.

🔷

TypeScript First

Written in TypeScript with complete type definitions. IntelliSense for all props, slots, and events. Generic component support.

Installation

Install Nexus UI and its peer dependencies using your preferred package manager:

npm
yarn
pnpm
bun
Terminal 
npm install @nexus-ui/react @nexus-ui/theme

Quick Start

Wrap your application with the NexusProvider and start using components:

App.tsx 
import { NexusProvider, Button, Input } from '@nexus-ui/react'
import { defaultTheme } from '@nexus-ui/theme'

export default function App() {
  return (
    <NexusProvider theme={defaultTheme}>
      <Input
        label="Email address"
        type="email"
        placeholder="you@example.com"
        variant="outlined"
      />
      <Button color="primary" size="lg">
        Get Started
      </Button>
    </NexusProvider>
  )
}
ℹ️
The NexusProvider component must wrap your entire application. It provides theme context, manages color mode, and handles portal containers for overlay components.

Configuration

Nexus UI can be configured through the provider's config prop. Here are the available configuration options:

theme Theme Required
The theme object containing design tokens, component styles, and color palette definitions.
colorMode 'light' | 'dark' | 'system'
Initial color mode. When set to 'system', automatically matches the user's OS preference.
Default: 'system'
locale string
BCP 47 locale identifier for internationalization. Affects date formatting, number formatting, and RTL detection.
Default: 'en-US'
portalContainer HTMLElement
Custom DOM element for rendering portal content (modals, drawers, tooltips). Useful for shadow DOM or iframe scenarios.
Default: document.body
disableAnimations boolean
Globally disable all component animations. Automatically true when prefers-reduced-motion is set.
Default: false

Framework Guides

Nexus UI works with all major React frameworks. Follow the specific guide for your setup:

Next.js (App Router)

Add the provider to your root layout. Mark it as a client component with 'use client'. Import the CSS in your global stylesheet.

Vite + React

Import the provider in your main entry file. No additional configuration needed. Tree-shaking works automatically with Vite's Rollup-based build.

Remix

Add the provider to your root route. Use the links export to include the CSS file. Server-side rendering is fully supported.

Astro

Use the React integration and add the provider to your layout component. Set client:load on interactive component islands.
